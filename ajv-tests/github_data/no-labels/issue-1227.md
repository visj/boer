# [1227] Ajv website and documentation

The number of Ajv features makes the documentation difficult to navigate.

Ajv would massively benefit from having a better structured web-site and documentation, and more user-friendly examples of features.

This issue can be done by somebody who has:
- an experience of technical writing and structured presentation of the information
- an interest to learn JSON Schema and Ajv
- web-design/development experience

It would probably take a week of work and there can be a [disproportionally] small incentive from Ajv open-collective (it has to be agreed in advance) - the main benefit would be from the opportunity to learn Ajv, so it can be a good project for somebody learning JSON Schema etc.

Please reach out to me if you are interested.
