**Are you going to resolve the issue?**
When search the source code, know the warning is from ajv.js
```
function _get$Id(schema) {
  if (schema.id) this.logger.warn('schema id ignored', schema.id);
  return schema.$id;
}
```
I also checked the issue of [521](https://github.com/epoberezkin/ajv/issues/521), but I do not have any option of it. According to issue 521, it should be fixed in 5.1.6.

The Ajv documentation mentions,

> schemaId: this option defines which keywords are used as schema URI. Option value:
"$id" (default) - only use $id keyword as schema URI (as specified in JSON Schema draft-06/07), ignore id keyword (if it is present a warning will be logged).

In my understand that the `schemaId` is for the schema URI, but right now there is `id` in the properties, which should not be relevant (?)
