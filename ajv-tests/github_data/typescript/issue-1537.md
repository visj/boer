# [1537] Missing propagation of second type parameter in JTDSchemaType

Hi! 👋 
      
Firstly, thanks for your work on this project! 🙂

I'm using:

- `ajv`: `8.0.5`
- `typescript`: `4.2.3`

Today I used [patch-package](https://github.com/ds300/patch-package) to patch `ajv@8.0.5` for the project I'm working on.

I have the following types:

```typescript
interface ConfigDataExtend {
  source?: 'github'
  version: string
}
export interface ConfigData {
  extends?: Record<string, ConfigDataExtend>
  rules?: string[]
}
```

After a bit of trial and error I arrived at the following schema:

```typescript
const schema: JTDSchemaType<ConfigData, { extendItem: ConfigDataExtend }> = {
  definitions: {
    extendItem: {
      optionalProperties: { source: { enum: ['github'] } },
      properties: {
        version: { type: 'string' },
      },
    },
  },
  optionalProperties: {
    extends: {
      values: {
        ref: 'extendItem',
      },
    },
    rules: { elements: { type: 'string' } },
  },
}
```

But here is the error I had on the `ref: 'extendItem'` line:

> Type '{ ref: string; }' is not assignable to type 'JTDSchemaType<ConfigDataExtend, Record<string, never>>'.
  Object literal may only specify known properties, and 'ref' does not exist in type 'JTDSchemaType<ConfigDataExtend, Record<string, never>>'.ts(2322)
jtd-schema.d.ts(59, 5): The expected type comes from property 'values' which is declared here on type 'JTDSchemaType<Record<string, ConfigDataExtend>, { extendItem: ConfigDataExtend; }>'
