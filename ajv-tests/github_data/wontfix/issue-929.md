**Are you going to resolve the issue?**

Yea, this is pretty straightforward.  I claim the RFC covering URLs doesn't require that the host component be a fully qualified name, in fact it doesn't even require that it is a DNS name (though it typically is);  [here's a link](https://tools.ietf.org/html/rfc3986#section-3).  I see that you're thinking about removing URL and other formats alltogether and refactoring them into a plugin (see #907), that's fine, but the plugin should also validate non-FQN, and until then so should ajv.

I've attempted to fix the URL regexes (both the one in use as well as the commented-out form, here they are, though I can make a pull request too and we can discuss here.  It's basically the same thing with an additional `?` to make the trailing TLD optional.
